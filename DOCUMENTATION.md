# FindAJob - Пълна техническа и проектна документация

## 1. Резюме

**FindAJob** е цялостно решение за уеб платформа за обяви за работа и професионална мрежа, проектирана да свързва работодатели с потенциални служители. Този документ служи като изчерпателна техническа справка, обхващаща архитектурата на системата, схемата на базата данни, дизайна на API, клиентското приложение, протоколите за сигурност и стратегиите за внедряване.

Платформата предлага не само механизми за публикуване и кандидатстване за работа, но и разширени социални функции, като потребителски профили, система за приятелства, съобщения в реално време и известия.

### 1.1 Цели на проекта
- **Ефективен процес на подбор:** Инструменти за работодателите за управление на обяви и кандидати.
- **Подобрено потребителско преживяване:** Възможност за служителите да създават богати профили, да качват CV-та и да следят кандидатурите си.
- **Социална свързаност:** Професионални връзки чрез вградена система за съобщения и приятелства.
- **Административен контрол:** Панел за управление на потребители, модерация на обяви и одобрение на регистрации.

### 1.2 Технологичен стек
- **Backend:** ASP.NET Core 10.0, C#, Entity Framework Core, SQLite
- **Frontend:** React 18, TypeScript, Vite, Vanilla CSS
- **Автентикация:** ASP.NET Core Identity (Cookie-based auth)

## 2. Схема на базата данни

Базата данни на FindAJob е проектирана с висока степен на нормализация и производителност.

### 2.1 Таблици за идентичност (Managed by ASP.NET Core Identity)
*   **AspNetUsers:** Основна таблица за потребителски акаунти.
*   **AspNetRoles:** Дефинира ролите в системата (`Admin`, `Employer`, `Employee`).

### 2.2 Основни таблици на приложението
*   **JobPostings (Обяви за работа):** Съдържа информация за заглавие, описание, локация, заплата, тип заетост и др.
*   **JobApplications (Кандидатури):** Проследява статуса на кандидатурата на даден потребител за конкретна обява.
*   **UserProfile (Потребителски профили):** Съдържа био, аватари и банери.

### 2.3 Социални функции
*   **Messages:** Вътрешна комуникация между потребителите.
*   **Friendships / FriendRequests:** Управление на професионални мрежи.
*   **Notifications:** Асинхронна система за известия.


# Част 1: Изследване на проблема и наличните решения

## 1.1 Проучване на съществуващи платформи за работа

Направихме подробен анализ на лидерите в индустрията: **Jobs.bg**, **Indeed** и **LinkedIn**.

### Jobs.bg
*   **Предимства:** Огромна потребителска база, локализация за българския пазар.
*   **Недостатъци:** Остарял интерфейс, липса на модерни социални функции.

### LinkedIn
*   **Предимства:** Богати професионални профили, отлични възможности за мрежово общуване.
*   **Недостатъци:** Висока цена за работодателите, сложни настройки за поверителност.

## 1.2 Дефиниране на нови функционалности за FindAJob

Въз основа на анализа, FindAJob се фокусира върху:
1.  **Вградена чат система:** Позволява директна комуникация без нужда от външни имейли.
2.  **Професионални приятелства:** Улеснява нетуъркинга в рамките на платформата.
3.  **Система за известия:** Информира потребителите за нови съобщения и промени в статуса на кандидатурите.

# Част 2: Проектиране на системата

## 2.1 Архитектура и комуникация

Системата използва модела **Client-Server** (Decoupled Single Page Application).
- **Клиент (Frontend):** Изграден с React 18 и TypeScript. Комуникацията се осъществява чрез REST API.
- **Сървър (Backend):** ASP.NET Core API, който обработва бизнес логиката и достъпа до данни.

## 2.2 Компоненти на проекта (Entities, Views, Models, Controllers)

- **Entities:** ApplicationUser, JobPosting, JobApplication, UserProfile, Message.
- **Views:** Начална страница, Търсене, Детайли за обява, Профил, Админ панел.
- **Controllers:** JobsController, ApplicationController, ProfilesController, AdminController.

# Част 3: Подбор на технологии

## 3.1 База данни и ORM

Проектът използва **Entity Framework Core (EF Core)** с подхода **Code-First**.
*   **SQLite:** Използва се за разработка и прототипиране поради своята лекота и преносимост.
*   **Seeding:** Автоматично генериране на примерни данни (компании като Google, Microsoft) за тестване на системата.

## 3.2 Backend и Frontend технологии

*   **Backend:** ASP.NET Core 10.0 – осигурява висока производителност и сигурност.
*   **Frontend:** React с Vite – за бързо зареждане и модерно потребителско преживяване.

# Част 4: Проектиране и изграждане на сървърна част

## 4.1 Управление на основната логика

### 4.1.1 Управление на обяви
Реализирано чрез `JobService`. Поддържа пълни CRUD операции и система за тагове.

```csharp
// Метод за създаване на обява
public async Task CreateJobAsync(JobPosting job)
{
    using var context = await _factory.CreateDbContextAsync();
    context.JobPostings.Add(job);
    await context.SaveChangesAsync();
}
```

### 4.1.2 Автоматични известия
Системата автоматично генерира известия при:
- Ново съобщение.
- Нова покана за приятелство.
- Промяна в статуса на кандидатура за работа.
