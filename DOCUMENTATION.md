# FindAJob - Пълна техническа и проектна документация

## 1. Резюме и въведение

**FindAJob** е модерна, пълнофункционална уеб платформа от корпоративен клас, проектирана да служи като мост между работодателите и търсещите работа. В свят, в който дигиталното присъствие е от решаващо значение за кариерното развитие, FindAJob предлага интуитивен интерфейс, съчетан с мощна сървърна логика, за да улесни процеса на подбор на персонал.

Този документ представлява изчерпателна „Пояснителна записка“, която детайлно описва всеки аспект от жизнения цикъл на разработката на софтуера – от първоначалното проучване на пазара до финалното тестване и оптимизация.

### 1.1 Цели на проекта
Основната цел на проекта е създаването на надеждна екосистема, която не само позволява публикуването на обяви, но и насърчава професионалното общуване.
- **Автоматизация на подбора:** Инструменти за филтриране и управление на десетки кандидатури едновременно.
- **Професионална идентичност:** Възможност за потребителите да поддържат онлайн портфолио, което включва образование, опит и умения.
- **Сигурност на данните:** Прилагане на съвременни стандарти за защита на личната информация и сигурна автентикация.

### 1.2 Целева аудитория
1.  **Служители (Кандидати):** Индивиди, търсещи нови кариерни възможности, които искат да представят своите умения по професионален начин.
2.  **Работодатели (Рекрутъри):** Компании и HR специалисти, нуждаещи се от платформа за лесно управление на обяви и бърз преглед на кандидати.
3.  **Администратори:** Оператори на платформата, отговорни за качеството на съдържанието и сигурността на системата.

### 1.3 Технологичен стек (Tech Stack)
Избраният стек е базиран на най-модерните индустриални стандарти за 2026 година:
- **Backend:** ASP.NET Core 10.0 (C#) – осигуряващ стабилност и висока производителност.
- **Frontend:** React 18 с TypeScript – за създаване на динамичен и типизиран потребителски интерфейс.
- **База данни:** SQLite с Entity Framework Core – за лесна преносимост и бързина при разработка.
- **Инструменти:** Vite (Build tool), Axios (API комуникация), xUnit (Тестване).

## 2. Схема на базата данни (Архитектура на данните)

Дизайнът на базата данни следва принципите на релационния модел и е проектиран за висока степен на разширяемост.

### 2.1 Таблици за потребители (ASP.NET Core Identity)
Използваме вградената система на Microsoft за управление на идентичността:
*   **AspNetUsers:** Съхранява основни данни за акаунта (имейл, хеширана парола, телефон).
*   **AspNetRoles:** Дефинира нивата на достъп: `Admin`, `Employer` (Работодател), `Employee` (Служител).
*   **AspNetUserRoles:** Свързваща таблица за много-към-много връзка между потребители и роли.

### 2.2 Основни бизнес таблици
*   **JobPostings:** Основната таблица за обяви. Включва заглавие, описание, локация, заплата, тип заетост (Remote/Hybrid) и флаг за „меко“ изтриване (`IsDeleted`).
*   **JobApplications:** Съхранява информация за това кой потребител за коя работа е кандидатствал, включително мотивационно писмо и текущ статус (Pending, Accepted, Rejected).
*   **UserProfile:** Разширена информация за потребителя – био, линкове към аватари и професионални титли.
*   **CvDocuments:** Пътища към качения PDF файл на автобиографията на потребителя.

### 2.3 Социален и комуникационен слой
*   **Messages:** Записи на всички съобщения в реално време, включително подател, получател и съдържание.
*   **Friendships:** Дефинира професионалните връзки между потребителите в платформата.
*   **Notifications:** Съхранява системни известия за нови съобщения, промени в статуса на кандидатури или нови покани за приятелство.

#### Технически пример: Модел на обявата
```csharp
// backend/Models/JobPosting.cs
public class JobPosting
{
    public int Id { get; set; }
    public string Title { get; set; } = string.Empty;
    public string Company { get; set; } = string.Empty;
    public string Salary { get; set; } = string.Empty;
    public string JobType { get; set; } = "Full-time"; // Тип заетост
    public string WorkMode { get; set; } = "Remote"; // Режим на работа
    public bool IsDeleted { get; set; } // Поддръжка на софт-делейт
    public string OwnerId { get; set; } = string.Empty; // Връзка към създателя
}
```

# Част 1: Изследване на проблема и наличните решения

## 1.1 Анализ на съществуващи платформи

За да създадем конкурентен продукт, анализирахме три основни платформи:

### Jobs.bg
Това е пазарният лидер в България. 
- **Силни страни:** Огромна база от обяви и разпознаваемост на марката. 
- **Слаби страни:** Дизайнът е остарял, липсват инструменти за директна комуникация в реално време, а потребителските профили са ограничени.

### Indeed
Глобален агрегатор на работни места.
- **Силни страни:** Изключително мощна търсачка и голям обем информация.
- **Слаби страни:** Често се срещат дублирани или неактуални обяви, а интерфейсът може да бъде претрупан.

### LinkedIn
Професионална социална мрежа.
- **Силни страни:** Отлични възможности за нетуъркинг и богати профили.
- **Слаби страни:** Много висока цена за работодателите и прекалено много „шум“ (непрофесионално съдържание) във фийда.

## 1.2 Нови функционалности в FindAJob

Въз основа на горепосочения анализ, внедрихме следните иновации:
1.  **Интегрирана чат система:** Премахва нуждата от преминаване към външни имейл платформи.
2.  **Система за професионални приятелства:** Позволява изграждането на доверена мрежа от контакти.
3.  **Автоматични статус-известия:** Потребителят веднага разбира, когато неговата кандидатура е прегледана или статусът ѝ е променен.

# Част 2: Проектиране на системата

## 2.1 Архитектурен модел

Системата е изградена като **Decoupled Single Page Application (SPA)**:
- **Frontend (React):** Отговаря за визуализацията и потребителското преживяване. Комуникира със сървъра чрез JSON пакети.
- **Backend (ASP.NET Core):** Изпълнява ролята на защитен шлюз към базата данни и обработва цялата бизнес логика.

## 2.2 Основни компоненти

### 2.2.1 Обекти (Entities)
Дефинирахме над 10 основни обекта, включително `ApplicationUser`, `JobPosting`, `JobApplication`, `Message`, `Notification` и др.

### 2.2.2 Изгледи (Views/Pages)
1.  **Начална страница:** Търсене и филтриране на обяви.
2.  **Страница на обявата:** Детайлно описание и бутон за кандидатстване.
3.  **Профил на потребителя:** Секции за опит и умения.
4.  **Административен панел:** Управление на цялата система.
5.  **Съобщения:** Интерфейс за чат.

### 2.2.3 Контролери (Controllers)
API-то е разделено на логически сектори: `JobsController` (обяви), `AccountController` (акаунти), `AdminController` (администрация).

# Част 4: Проектиране и изграждане на сървърна част

## 4.1 Бизнес логика и услуги (Services)

Сървърната част е организирана чрез **Service Layer** модела. `JobService` е централният компонент, който обработва сложните заявки към базата данни.

#### Пример за логика в JobService:
```csharp
// backend/Services/JobService.cs
public async Task<List<JobPosting>> GetJobsAsync()
{
    using var context = await _factory.CreateDbContextAsync();
    
    // Извличаме само активни обяви (IsDeleted = false)
    return await context.JobPostings
        .Include(j => j.JobPostingTags)
        .ThenInclude(jt => jt.Tag)
        .Where(j => !j.IsDeleted)
        .OrderByDescending(j => j.CreatedAt)
        .ToListAsync();
}
```

## 4.2 Автоматични известия
Внедрихме логика, която при всяко ключово действие (напр. ново съобщение) автоматично създава запис в таблицата `Notifications` за съответния потребител.
